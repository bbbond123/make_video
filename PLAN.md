# AI 视频批量生产系统 - 项目计划

## 项目目标

构建一个自动化视频生产流水线，实现：
- 输入：文案
- 输出：每天 3 条 AI 虚拟人物视频（30-60秒）
- 平台：全平台通用（抖音、小红书、视频号、快手等）

---

## 一、视频形态（三选一或组合）

| 形态 | 描述 | 复杂度 | 适用场景 |
|-----|------|--------|---------|
| AI真人口播 | 固定人物正面说话，口型同步 | 中 | 信任感强，适合观点输出 |
| 人物+场景 | 人物在生活场景中，配旁白 | 中 | 情绪/感悟类，容错率高 |
| 定格+字幕 | 人物轻微动作，重点靠字幕配音 | 低 | 批量最快，稳定 |

**建议**：先从「定格+字幕」或「AI真人口播」起步

- ✅ 无需 GPU
- ✅ 国内可用（n8n 自托管）
- ✅ 支持 Cameo（人物一致性）
- ✅ 零配置部署，一键包
- ✅ 批量化、自动化
- ✅ 无水印输出

| 版本 | 720P | 1080P | 适用场景 |
|-----|------|-------|---------|
| Sora 2 | $0.10/秒 | - | 创意探索、快速测试 |
| Sora 2 Pro | $0.30/秒 | $0.50/秒 | 专业级画质 |

**成本估算（每天3条，每条30秒）：**
- Sora 2 (720P): 3 × 30 × $0.10 = **$9/天** ≈ ¥65/天

## 四-B、扩展研究（详细）

### 1. Cameo 角色一致性功能

> 来源：[Sora2 Cameo API教程](https://www.cnblogs.com/grsai/p/19234635)

**什么是 Cameo：**
- 上传个人验证视频，提取面部特征、体型和声音特征
- 将自己/朋友/宠物/原创角色放入 AI 生成的视频中
- 在不同镜头切换时保持角色外貌、衣着一致

**使用方法：**
```
1. 创建 Cameo 形象（录制短视频）
2. 在提示词中加入 cameo:姓名 标记
3. 或使用 @用户ID 调用公开 Cameo
```

**API 参数要求：**
- 格式：MP4，公开访问 URL
- 时长：最长 3 秒
- 时间戳格式：`start_second,end_second`
- 提示词语法：`@character` + 位置索引

**价格：**
- GrsAi 中转：**¥0.08/条**（史低价）

**注意事项：**
- Cameo 请求需 24 小时内处理
- 保持描述一致性，避免混合冲突特征
- 原版需美国/加拿大 IP + 邀请码

### 4. n8n 部署方案

> 来源：[n8n中文版安装指南](https://zhuanlan.zhihu.com/p/1910426360054285764)

**部署方式：**

| 方式 | 适合人群 | 难度 |
|-----|---------|------|
| npm 全局安装 | Node.js 开发者 | 中 |
| Docker run | 大多数用户（推荐） | 低 |
| Docker Compose | 最佳实践 | 低 |
| 钞能力一键包 | Windows 小白 | 最低 |

**Docker Compose 示例：**
```yaml
version: '3'
services:
  n8n:
    image: n8nio/n8n
    ports:
      - "5678:5678"
    volumes:
      - ~/.n8n:/home/node/.n8n
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=your_password
```

**中文汉化：**
- 下载汉化包
- 修改配置文件（添加 zh-cn 路径）
- 重启容器

## 附录：原始需求文档

### A. 整体方案概览

项目定位：**「AI 虚拟人物 + 文案驱动的视频工厂」**

核心结构：
> **文案 → AI 人物 → 配音 → 场景 → 自动剪辑 → 批量输出**

每天 3 条 ≈ **一套模板 + 自动替换内容**

---

### B. 视频形态详细说明

#### 形态 1：AI 真人口播（最稳）

- 一个"固定人物形象"
- 正面或半身
- 对着镜头说生活感悟 / 日常经验

**适合平台**：抖音 / 快手 / 视频号 / 小红书

**优势**：
- 不需要拍摄
- 情绪、信任感强
- 非常适合生活类内容

#### 形态 2：人物 + 生活场景（轻剧情 / 旁白）

- 人在走路、咖啡馆、街道、家中
- 文案作为旁白或字幕

**优势**：
- 不用口型严格对齐
- 容错率高
- 适合感悟 / 情绪 / 日常分享

#### 形态 3：人物定格 + 动态字幕（最低成本）

- 人物轻微动作（眨眼、呼吸）
- 重点靠字幕和配音

**优势**：
- 稳定、批量最快
- 很适合一天多条
